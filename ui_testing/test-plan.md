# Test Plan - Sauce Demo

## Objective
Validate the main authentication functionalities of the login system, listing and sorting of products, the purchase flow of products, cart product removal, pages navigation and logout.

---

## Scope

### Features to be tested
- User login
- Input validation
- Error messages
- Logout functionality
- Products purchase
- Products Catalog (listing, sorting and pagination)

---

## Test Environment

| Item | Details |
|---|---|
| Application URL | https://www.saucedemo.com |
| Browser | Google Chrome |
| Operating System | Windows 10 |
| Test Type | Manual Testing |

---

## Test Strategy

The tests will be executed manually through the user interface, focusing on:
- Positive scenarios
- Negative scenarios
- Navigation flow

---

## Risks

| Risk | Impact |
|---|---|
| Weak validation messages | Poor user experience |
| Login bypass possibility | Critical security risk |

---

## Entry Criteria
- Application accessible
- Test credentials available

---

## Exit Criteria
- All critical scenarios executed
- Major defects documented
- Evidence captured

---

## Deliverables
- Test cases
- Bug reports
- Screenshots
- Final test results
